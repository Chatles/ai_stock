# 公告智能分析系统 - 架构与需求文档

> **最终目标**：构建一个基于A股公告的事件驱动投资辅助系统，通过AI深度分析公告内容，预测对次日股价的影响，辅助投资者做出决策。

---

## 一、项目概述

### 1.1 核心价值
- **快速筛选**：从海量公告中快速识别对股价有明显影响的公告
- **深度分析**：AI分析公告内容，预测次日涨跌概率和幅度
- **信息整合**：整合公司基本面、行业情况、竞争格局等多维度信息
- **持续优化**：通过历史回溯分析，不断提升预测准确率

### 1.2 用户场景
> "我每天收盘后需要一个工具，帮我快速看到当天发布的所有公告中，哪些可能在明天对我持有的股票或我关注的股票产生正面影响。我需要知道：
> 1. 这份公告的主要内容是什么
> 2. 这家公司现在的估值和基本面如何
> 3. 它在行业中处于什么地位
> 4. 竞争对手近期有什么动作
> 5. 基于以上信息，明天股价上涨的概率有多大，预计涨多少"

CREATE TABLE notice_analysis (
  id INTEGER PRIMARY KEY,
  notice_id TEXT UNIQUE,              -- 公告唯一ID

  security_name TEXT,                 -- 股票名称

  notice_url TEXT,                    -- 公告链接

  industry_analysis TEXT,             -- 行业分析
  competitive_analysis TEXT,           -- 竞争分析
  analysis_reason TEXT,               -- 分析理由
  analyzed_at TEXT,                   -- 分析时间
  created_at TEXT                     -- 创建时间

### 3.2 现有分析流程

```
获取公告
    │
    ▼

    │                                │
    │ (含利好关键词)                 │
    ▼                                │

    │                                │
    ├── 利好(1-5分) ◄────────────────┘

```

### 5.1 数据源扩展

#### 5.1.1 免费数据源

| 数据类型 | 数据源 | 费用 | 说明 |

| 估值数据 | AKShare | 免费 | PE、PB、PS等 |

| 数据类型 | 数据源 | 费用 | 优先级 |

| 研报摘要 | 同花顺/东方财富 | 免费 | 中 |

| 优化项 | 策略 | 节省比例 |
|-------|------|---------|
| 标题过滤 | 规则匹配，无需AI | 100% |
| 分级分析 | 只对高价值公告用AI | ~70% |
| 批量获取 | 一次获取多个指标 | ~20% |
| 缓存策略 | 财务数据每日更新 | ~50% |
| 摘要提取 | 用廉价模型提取PDF摘要 | ~30% |

### 5.4 历史回溯优化系统

#### 5.4.1 回溯分析流程

```
                    ┌─────────────────────────┐
                    │   选择历史日期范围       │
                    │   (如: 过去30天)        │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   获取该期间所有公告     │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   执行标准分析流程       │
                    │   (标题过滤 → AI分析)   │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   获取次日真实涨跌数据   │
                    │   (Tushare/AKShare)     │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   对比预测 vs 实际       │
                    │   计算准确率             │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   统计分析               │
                    │   - 利好公告次日上涨率   │
                    │   - 不同利好程度准确率   │
                    │   - 误差分析             │
                    └───────────┬─────────────┘
                                │
                    ┌───────────▼─────────────┐
                    │   优化模型               │
                    │   - 调整关键词权重       │
                    │   - 优化AI Prompt       │
                    │   - 更新分析阈值         │
                    └─────────────────────────┘
```

#### 5.4.2 准确率评估指标

| 指标 | 计算方式 | 目标 |
|-----|---------|------|
| 整体准确率 | 预测正确数 / 总数 | >60% |
| 利好召回率 | 实际利好被识别 / 实际利好总数 | >70% |
| 精确率 | 识别为利好的正确率 | >50% |
| 涨跌幅误差 | 预测值与实际值偏差 | <2% |

## 七、技术债务与风险

### 7.1 当前风险

| 风险 | 影响 | 应对 |
|-----|------|------|
| Token消耗过快 | 成本 | 分层分析+缓存 |

| AI幻觉 | 准确性 | 回溯验证+人工审核 |

### 7.2 技术债务

| 债务 | 说明 | 建议 |
|-----|------|------|
| 缺少单元测试 | 测试覆盖不足 | 添加Jest |
| 类型定义不完整 | TypeScript优势未发挥 | 补充类型 |
| 错误处理不统一 | 部分API缺少容错 | 全局ErrorHandler |
| 日志不完善 | 问题排查困难 | 结构化日志 |
